# Push

変更されているファイルを確認し、品質チェックを通してからコミット・プッシュしてください。

手順:

1. `make format` で整形する
2. `make lint` で静的解析を通す
3. `make test` でテストを通す
4. `git diff` と `git status` で変更内容を把握する
5. 関連するファイルをグループ化し、1グループ＝1コミットとする
6. 各グループを `git add <files>` → `git commit` で順番にコミットする
7. すべてのコミットが終わったら `git push` する

コミットメッセージは各変更の内容から適切に生成してください。
